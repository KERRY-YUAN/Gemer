# Gemer — 智能对话与本地系统接管桥梁

Gemer 是一款创新的浏览器扩展与本地物理通道中转工具（基于 Chrome Native Messaging API）。它旨在打破传统浏览器沙箱的限制，让网页端的 AI 助手（如 ChatGPT、Gemini、Google AI Studio）在获得您授权的前提下，能够安全、便捷地读取和控制您的本地计算机系统，实现“探讨方案-生成指令-本地执行-数据回传”的闭环自动化体验。

> **扩展程序版本 (Manifest)**: v0.4.2

---

## 🌟 核心特性

- **零端口占用与无感中转**：不启动本地 HTTP 服务器，不占用本地网络端口，利用 Chrome 官方底层标准输入输出（Stdio）进行双向安全通讯，规避本地防火墙和网络策略拦截。
- **全自动生命周期托管**：本地后台守护进程完全由浏览器生命周期托管。
  - **随展随启**：在浏览器页面中展开控制面板时，Chrome 会自动拉起本地 Python 进程。
  - **随关随停**：收起面板、刷新网页或关闭标签页时，Chrome 会自动断开并销毁后台进程，不占用任何系统残留资源。
- **高精度指令解析引擎**：采用改良后的防跨越式前瞻否定断言正则，精确提取页面中最新生成的 `[CMD_START] ... [CMD_END]` 控制指令，完全避免长对话中历史旧指令的二次重复触发。
- **多平台广泛兼容**：一套代码同时适配并兼容 **ChatGPT (chatgpt.com)**、**Gemini (gemini.google.com)** 以及 **Google AI Studio (aistudio.google.com)** 三大主流对话工作区。
- **一键配置与依赖自完备**：提供自启修复工具，首次安装仅需运行一次，即可在本地自动化构建完全隔离的 Python 绿色版运行环境与依赖，并完成注册表一键配置。

## 📂 项目结构

```text
Gemer/
├── com.gemer.bridge.json   # Chrome 宿主声明配置文件
├── Gemer.bat               # 静默唤醒免安装 Python 的引导脚本
├── Gemer.py                # 后台物理命令执行服务（不显示任何控制台黑框）
├── install_host.bat        # 【一键配置】自动下载绿色版 Python、补全依赖、并注册系统通道
├── requirements.txt        # 依赖项清单（仅包含轻量依赖）
├── sys/                    # 【新增】由安装脚本自动下载并部署的完全隔离的 Python 绿色版运行环境
└── gemer_extension/        # 浏览器插件源码目录 (Manifest v0.4.2)
    ├── manifest.json       # 权限与后台服务清单
    ├── background.js       # 通道中转服务进程（管理 Python 的启动与关闭）
    ├── content.js          # 前端交互与高保真输入模拟引擎
    └── styles.css          # 高颜值浮动控制面板样式表
```

## 🚀 安装与配置指引（一次性配置）

### 1. 准备本地环境
*无需在操作系统中预先安装配置 Python 环境变量。初始化工具会在运行后自动在本地部署独立的、完全隔离的绿色免安装版 Python 运行环境。*

### 2. 装载 Chrome 浏览器扩展
1. 打开 Chrome 浏览器，在地址栏输入并访问 `chrome://extensions/`。
2. 在右上角开启 **“开发者模式”** (Developer Mode)。
3. 点击左上角的 **“加载已解压的扩展程序”** (Load unpacked)，选择并导入本项目目录下的 `gemer_extension` 文件夹。
4. 加载完成后，**复制该扩展程序卡片上显示的“ID”**（一串类似于 `knldjmfmop...` 的 32 位英文字母）。

### 3. 一键初始化注册
1. 返回本地 `Gemer` 根目录，双击运行 `install_host.bat`。
2. 脚本将自动镜像服务器（华为云镜像源与 Gitee 镜像源）为您一键部署独立的 Python 绿色版运行环境并补全运行依赖。
3. 按照命令窗口提示，**粘贴刚才复制的“扩展程序 ID”**，然后按下回车。
4. 注册表和服务端配置将自动写入，提示配置成功后，您可以彻底关闭此文件夹。

> 💡 **关于杀毒软件拦截的声明**
>
> 运行 `install_host.bat` 时，由于脚本需要将配置文件路径写入注册表 `HKCU\Software\Google\Chrome\NativeMessagingHosts`，部分杀毒软件（如 Windows Defender、火绒等）可能会弹出拦截警告。这是系统对注册表修改行为的正常保护，请在提示中选择**“允许本次操作”**或**“信任/放行”**，否则扩展程序将无法唤醒本地 Python。

## 🎈 交互流程示例

1. 打开 **ChatGPT**、**Gemini** 或 **Google AI Studio**，您会在网页右下角看到一个精美的 **「✦ Gemer」** 按钮。
2. 点击按钮展开面板，后台通道将自动激活（提示：`🟢 本地物理通道已对接`）。
3. 点击 **「📝 注入系统设定」**，插件会自动将系统提示词送入输入框并提交（建立 AI 对话人设）。
4. 接下来，您可以像平常一样向 AI 提问（例如：`“帮我看看我的显卡型号，顺便检查一下 C 盘剩余空间”`）。
5. 当 AI 给出系统分析指令（如 `[CMD_START] wmic ... [CMD_END]`）时，面板上会弹出醒目的 **「批准执行」** 申请卡片。
6. 点击批准后，系统指令将静默在后台完成，执行结果（回执）将自动粘回网页并提交给 AI，由 AI 进行下一步方案规划，完成全流程闭环。

## ⚠️ 安全警示与免责声明

**请在运行本项目前仔细阅读并知悉以下安全规范：**

### 🚨 核心风险：防范“提示词注入攻击（Prompt Injection）”
当您允许 AI 助手阅读、分析未经审查的外部数据（例如：使用联网搜索功能、让 AI 总结特定网页、或者让 AI 读取您上传的未知 PDF/代码文件时），恶意第三方可能在网页或文档中隐藏欺骗性的指令。
* **攻击场景**：网页中隐藏了 `[CMD_START] 恶意命令 [CMD_END]`。AI 在阅读该网页后，可能会盲目遵照其指示输出该恶意指令，从而触发 Gemer 的弹窗。
* **安全建议**：**凡是在涉及“联网搜索”或“读取外部文件”的对话上下文中，如果 Gemer 面板弹出了物理执行申请，请务必保持极高警惕！** 仔细核对命令框中的每一行代码，若非您本人主动要求的操作，请立即点击「拒绝」。

### 🔒 物理权限控制
1. **指令复核**：本项目在执行前均设有弹窗确认提示。请在点击“批准执行”前，务必认真核对指令的具体内容。
2. **高危操作隔离**：严禁批准包含破坏性删除（如 `del /s`）、磁盘格式化（如 `format`）以及任何未知外部网络请求（如通过 `curl` 或 `wget` 下载并运行外部脚本）的指令。
3. **最小权限原则**：请以普通用户身份（非 Administrator 账户）运行浏览器，以确保即使执行了受信任命令，其权限范围也受到本地系统权限规则的严格约束。
4. **隐私保护说明**：执行命令所得的回执数据（如本地路径、硬件配置、运行状态等）将作为对话上下文发送给大语言模型服务端（如 OpenAI、Google）。请确保返回的数据中不包含您个人或企业的机密隐私数据。
