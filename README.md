# Gemer — 智能对话与本地系统接管桥梁

Gemer 核心实现了`网页端AI`与`本地执行`的`联通`。
它是一款基于 Chrome Native Messaging API 的浏览器扩展与本地中转工具，打破了传统浏览器沙箱限制，在用户明确授权下，实现网页端 AI （ChatGPT、Gemini、Google AI Studio）与本地系统之间的“指令下发-本地执行-数据回传”自动化联通闭环。

> **扩展程序版本 (Manifest)**: v0.4.3

---

##  核心特性

- **零端口无感中转**：基于 Chrome 官方底层标准输入输出（Stdio）通讯，不启动本地 HTTP 服务器，免受防火墙和网络策略拦截。
- **生命周期全托管**：本地 Python 进程随浏览器面板展开而自动拉起，随页面关闭而自动销毁，零系统资源残留。
- **精准长对话防误触**：采用前瞻否定断言正则提取最新 `[CMD_START] ... [CMD_END]` 指令，自动忽略历史旧指令。
- **多平台广泛兼容**：一套代码同时适配并兼容 **ChatGPT (chatgpt.com)**、**Gemini (gemini.google.com)** 以及 **Google AI Studio (aistudio.google.com)** 三大主流对话工作区。
- **多工作区与全隔离环境**：一键适配三大主流 AI 平台；内置一键初始化脚本，自动部署完全隔离的 Python 绿色运行环境，无需用户预装 Python。

##  项目结构

```text
Gemer/
├── com.gemer.bridge.json   # Chrome 宿主声明配置文件
├── Gemer.bat               # 静默唤醒免安装 Python 的引导脚本
├── Gemer.py                # 后台物理命令执行服务（不显示任何控制台黑框）
├── install_host.bat        # 【一键配置】自动下载绿色版 Python、补全依赖、并注册系统通道
├── requirements.txt        # 依赖项清单（仅包含轻量依赖）
├── sys/                    # 由安装脚本自动下载并部署的完全隔离的 Python 绿色版运行环境
└── gemer_extension/        # 浏览器插件源码目录
    ├── manifest.json       # 权限与后台服务清单
    ├── background.js       # 通道中转服务进程（管理 Python 的启动与关闭）
    ├── content.js          # 前端交互与高保真输入模拟引擎
    └── styles.css          # 高颜值浮动控制面板样式表
```

## 快速开始

### 0. 安装准备

将 `Gemer` 文件夹放置在本机某目录下。
*无需预装Python环境。初始化工具会在运行后自动在本地部署独立的、完全隔离的绿色免安装版 Python 运行环境。*

### 1. 安装与一键配置

1. **加载扩展**：打开 Chrome 进入 `chrome://extensions/` -> 开启**开发者模式** -> 点击**加载未打包的扩展程序** -> 选择 `gemer_extension` 文件夹 -> **复制生成的 32 位扩展程序 ID**。
2. **物理激活**：进入 `Gemer` 根目录，双击运行 `install_host.bat` -> 粘贴刚才复制的 **扩展程序 ID** -> 回车完成自动部署与注册表写入（如遇杀毒软件拦截，请选择“允许/放行”）。

### 2. 闭环使用流程

打开 **ChatGPT / Gemini / Google AI Studio** 任意窗口：
点击右下角 **「✦ Gemer」** 面板（连接成功） -> 点击 **「📝 注入系统设定」**（建立 AI 人设） -> 正常对话提问（如：“检查 C 盘剩余空间”） -> 弹窗复核 AI 指令 -> 点击 **「批准执行」**（或输入用户指令并执行） -> 后台静默执行并自动回传结果给 AI。

> 💡 **关于杀毒软件拦截的声明**
>
> 运行 `install_host.bat` 时，由于脚本需要将配置文件路径写入注册表 `HKCU\Software\Google\Chrome\NativeMessagingHosts`，部分杀毒软件（如 Windows Defender、火绒等）可能会弹出拦截警告。这是系统对注册表修改行为的正常保护，请在提示中选择**“允许本次操作”**或**“信任/放行”**，否则扩展程序将无法唤醒本地 Python。

##  安全警示与免责声明

**请在运行本项目前仔细阅读并知悉以下安全规范：**

###  核心风险：防范“提示词注入攻击（Prompt Injection）”
当您允许 AI 助手阅读、分析未经审查的外部数据（例如：使用联网搜索功能、让 AI 总结特定网页、或者让 AI 读取您上传的未知 PDF/代码文件时），恶意第三方可能在网页或文档中隐藏欺骗性的指令。
* **攻击场景**：网页中隐藏了 `[CMD_START] 恶意命令 [CMD_END]`。AI 在阅读该网页后，可能会盲目遵照其指示输出该恶意指令，从而触发 Gemer 的弹窗。
* **安全建议**：**凡是在涉及“联网搜索”或“读取外部文件”的对话上下文中，如果 Gemer 面板弹出了物理执行申请，请务必保持极高警惕！** 仔细核对命令框中的每一行代码，若非您本人主动要求的操作，请立即点击「拒绝」。

###  物理权限控制
1. **指令复核**：本项目在执行前均设有弹窗确认提示。请在点击“批准执行”前，务必认真核对指令的具体内容。
2. **高危操作隔离**：严禁批准包含破坏性删除（如 `del /s`）、磁盘格式化（如 `format`）以及任何未知外部网络请求（如通过 `curl` 或 `wget` 下载并运行外部脚本）的指令。
3. **最小权限原则**：请以普通用户身份（非 Administrator 账户）运行浏览器，以确保即使执行了受信任命令，其权限范围也受到本地系统权限规则的严格约束。
4. **隐私保护说明**：执行命令所得的回执数据（如本地路径、硬件配置、运行状态等）将作为对话上下文发送给大语言模型服务端（如 OpenAI、Google）。请确保返回的数据中不包含您个人或企业的机密隐私数据。
